How it works

Tier 1 — heals itself, no human. Reloads an unloaded job, kickstarts a stale one, re-runs a failed one (which clears transient upstream blips like the Todoist 503 that killed the concierge sweep), re-runs a failed workflow, restarts a dead daemon, prunes bloated logs. Capped at 3 attempts per system per day with a 20-minute cooldown, because a healer with no limits is just a slower outage.

Tier 2 — asks first. If tier 1 didn't fix it, it's a real bug, not a blip. Claude diagnoses it in an isolated git worktree, fixes it on a branch, and opens a PR. Nothing lands on main unattended, and sentinel never merges its own repairs.

Drift — coverage is enforced, not remembered. Any new launchd job or workflow file is auto-registered on the next pass with conservative defaults. "Remember to add it to the registry" is not a control, it's a hope, and it's one forgotten edit away from a blind spot.

Silence is the healthy signal. Telegram fires only when a human is actually needed. The morning brief cross-checks by flagging sentinel itself if it ever goes stale — because the one failure nobody would otherwise be told about is the monitoring going dark.
