🔐 Getting in: SSH vs VNC

Both ride over Tailscale (same private network on mini + phone + laptop → use the mini's Tailscale IP from anywhere, no router setup).

⌨️ SSH terminal

ssh kevin@<mini-ip>
→ Your daily door. 95% of everything.

🖱️ VNC the desktop

vnc://<mini-ip>
→ Only when you need to click something.
🧭 Rule: SSH by default, VNC when a mouse is required (GUI first-run dialogs, QR-code pairing). You'll rarely need VNC.

📱 Door #1 · Remote Control — code from your phone

The gist

A Claude Code session runs on the mini; your phone is the screen. Outbound HTTPS only — no open ports.

▶️ Use itClaude app → Code tab → tap "Mac mini" → you're in a live session
🔧 Runs vialaunchd agent → claude remote-control --name "Mac mini" --spawn=same-dir in ~/second-brain
✅ Needssigned into claude.ai · folder trusted: cd ~/second-brain && claude once, accept, /exit
🛠️ Full wrapper + launchd agent (copy-paste)

~/second-brain-scripts/claude-rc.sh

#!/bin/bash
set -uo pipefail
export PATH="/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin"
cd "$HOME/second-brain" || exit 1
exec claude remote-control --name "Mac mini" --spawn=same-dir

~/Library/LaunchAgents/com.you.claude-remote-control.plist — keys: RunAtLoad ✔, KeepAlive ✔, ThrottleInterval 30, EnvironmentVariables (PATH + HOME), log paths under ~/Library/Logs/. Then:

launchctl bootstrap gui/501 ~/Library/LaunchAgents/com.you.claude-remote-control.plist
launchctl list | grep claude-remote-control   # want a PID + 0

💬 Door #2 · Channels — text your mini

The gist

A Telegram bot = a persistent Claude session on the mini. Text it, it reads your stuff and texts back. Your calendar / Gmail / Todoist connectors are already wired in (it runs as your subscription).

🤖 Make a bot@BotFather → /newbot → save token → put it in ~/.claude/channels/telegram/.env
🔒 Lock it downallowlist only your own chat_id so strangers can't talk to it
▶️ Use itjust text the bot · channels alias to watch it live
✅ Verifyattach → header says Claude Max/Pro, not "API Usage Billing"
🛠️ Full wrapper script (the launchd→tmux→Claude chain)

~/second-brain-scripts/claude-channels.sh — launchd keeps the script alive → script keeps a tmux session alive → tmux keeps Claude alive.

#!/bin/bash
set -uo pipefail
export PATH="/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin"
SESSION="channels"
cd "$HOME/second-brain" || exit 1
if ! tmux has-session -t "$SESSION" 2>/dev/null; then
  tmux new-session -d -s "$SESSION" -c "$HOME/second-brain" \
    "env -u ANTHROPIC_API_KEY claude --model sonnet \
       --channels plugin:telegram@claude-plugins-official"
fi
while tmux has-session -t "$SESSION" 2>/dev/null; do sleep 30; done

Model pinned to sonnet to stretch plan limits. launchd plist is the same shape as Remote Control's.

🪟 tmux — the "stays running after you leave" trick

Do thisCommand
Watch a session livetmux attach -t channels (or the channels alias)
Leave it runningCtrl-b then d (detach)
List sessionstmux ls
Kill onetmux kill-session -t channels
🚫 The one mistake everyone makes: to leave it running you detach (Ctrl-b d). Typing exit or Ctrl-c kills it. Detach ≠ exit.

⚠️ The two gotchas that will bite

🔑 #1 — Start sessions via launchd, never by hand over SSH An SSH-launched claude can't unlock the keychain → can't read your claude.ai token → silently bills your API account. Same command under launchd shows "Claude Max." Restart with launchctl kickstart -k gui/501/<label>, never a manual run.
🤖 #2 — One bot = one poller Running any claude over SSH that loads the telegram plugin (even claude mcp list) starts a 2nd poller and kills your live bot. To peek from SSH: read logs or tmux attach — don't launch a fresh claude. Fix: attach → /mcp → telegram ✗ failed → Reconnect.

🛰️ Nice trick: every session knows where it is

A SessionStart hook prints which machine + how you connected (phone-via-Remote-Control vs SSH) straight into Claude's context — so it adapts (e.g. knows the mini doesn't auto-push git, and reminds itself to push).

🛠️ How the hook works + the script

Wire a SessionStart command hook in .claude/settings.json to a script that (1) reads hostname → friendly name, (2) walks the parent-process chain for remote-control to tell phone from SSH, (3) prints JSON Claude treats as context:

host="$(hostname -s)"
case "$host" in *Mac-mini*) device="Mac mini" ;; *Air*) device="MacBook Air" ;; esac
# walk parents for 'remote-control' → transport = phone vs SSH
printf '{"hookSpecificOutput":{"hookEventName":"SessionStart",'
printf '"additionalContext":"Device: %s · %s"}}\n' "$device" "$transport"
